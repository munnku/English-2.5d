# 英語学習 3Dトップダウンゲーム プロジェクトリサーチレポート

本レポートでは、提案された「トップダウン視点英語学習ゲーム」を個人開発でWebアプリケーションとして実現するための、技術スタック、ビジネスモデル、セキュリティ（決済・認証）、および個人事業としてのベストプラクティスについて詳細にまとめています。

---

## 1. 技術スタックと推奨サービス

Webブラウザ上で動作する3Dゲームを開発し、ユーザー管理や決済を統合するための技術スタックです。

### ゲームエンジン / 3D描画
個人開発でのWeb向け3Dゲームの場合、ファイルサイズとWeb技術との親和性が重要になります。

*   **推奨1: React Three Fiber (Three.js)**
    *   **特徴**: Reactの文法でThree.jsを扱えるライブラリ。Webフロントエンド（UI設計など）との親和性が非常に高く、Reactエコシステムをフル活用できるため、現在最もモダンで人気があります。
    *   **適正**: ゲーム要素（移動や当たり判定など）もライブラリ（`@react-three/rapier`など）を組み合わせることでトップダウンRPG程度なら十分に構築可能です。
*   **推奨2: Godot Engine (Webエクスポート)**
    *   **特徴**: 軽量で扱いやすいOSSのゲームエンジン。バージョン4からはWebエクスポートもサポートされています。
    *   **適正**: より「本格的なゲーム開発」のワークフローを好む場合におすすめですが、Next.jsなどのモダンWebフレームワークとの統合（UIとの連携など）は少し工夫が必要です。

### Webフロントエンド＆バックエンド (BaaS)
*   **フロントエンドフレームワーク: Next.js (React) または Vite (React)**
    *   ゲーム以外のUI（登録画面、ダッシュボード、課金画面）の作成に最適です。
*   **バックエンド / データベース: Supabase または Firebase**
    *   ユーザー登録（Authentication）や学習データの保存（Database）を自前でサーバーを構築せずに実現できる「BaaS」を使用するのが個人開発の鉄則です。特にSupabaseはPostgreSQLベースで使いやすくお勧めです。
*   **ホスティング（デプロイ先）: Vercel または Cloudflare Pages**
    *   フロントエンドのホスティングとして非常に優秀。独自ドメインの設定も無料・数クリックで完了し、SSL化（HTTPS化）も自動で行われます。

## 2. ビジネスモデル（マネタイズ戦略）

教育系ゲーム事業を個人で展開する場合の現実的なビジネスモデルです。

*   **フリーミアム（一部無料＋買い切り/サブスク）**
    *   **概要**: 最初の1エリア（または基本の日常会話レベル）までは完全に無料でプレイでき、その後のステージや「応用文法」「旅行英会話パック」などを有料にするモデル。
    *   **メリット**: Webゲームの最大の強みである「URLを踏むだけですぐ遊べる（離脱率が低い）」というメリットを活かしてユーザーを集めやすくなります。
*   **サブスクリプション（月額課金）**
    *   **概要**: 月額500円〜1000円程度で、全ての機能と言語パックにアクセスできるモデル。定期的に新しいコンテンツ（シナリオ、新しい街、イベント）を追加できる体制が作れるなら最も収益が安定します。
*   **アイテム課金（コスメティック・時短）**
    *   **概要**: 学習を少し有利にするアイテムパックや、宇宙人の着せ替えスキンなどを販売。教育ゲームでは主要な収益源になりにくいですが、サブスク等の補助として機能します。

**★おすすめライン**: 「無料お試しエリア」＋「完全版ロック解除（買い切り）」が、個人開発者が最初に運用・保守する上で最も負担が少ない形です。

## 3. セキュリティ：ユーザー登録・決済サービスの脆弱性と対策

ウェブアプリにおいて、認証と決済のセキュリティは最も重要です。**「自作しない（車輪の再発明をしない）」**のが最大の対策です。

### 認証（ユーザー登録・ログイン）
*   **利用サービス**: Supabase Auth, Firebase Auth, Auth0 など
*   **脆弱性と対策**:
    *   **パスワードの漏洩**: 自作DBでパスワードを平文保存してしまうミス。上記サービスを使えば強固にハッシュ化されるため対策済みになります。
    *   **XSS / CSRF攻撃**: フロントエンドのフレームワーク(Next.js/React)の標準機能によりXSSは基本防御されます。セッション管理もBaaSのSDKに任せることで安全に実装できます。

### 決済サービス
*   **利用サービス**: **Stripe (Stripe Checkout)** または **Paddle**
    *   法人化していなくても個人事業主として導入可能で、クレジットカード情報を自社サーバーで一切保持しない仕組み（PCI DSS準拠）が簡単に作れます。
*   **脆弱性と対策**:
    *   **クレジットカード情報の漏洩**: StripeのCheckout（決済ページをStripe側に任せる機能）を利用すれば、アプリ側にカード情報が通過しないためリスクをほぼゼロにできます。
    *   **課金回避（Webhookの偽装）**:
        *   **脆弱性**: クライアント側（ブラウザ側）からの「決済完了」という信号だけを信じて有料アイテムを付与してしまうと、通信を改ざんされて無料でアイテムを取得されます。
        *   **対策**: 決済完了後、Stripeのサーバーから自分のサーバーへ送られる「Webhook」を受け取り、その署名を確認してからデータベース上でユーザーを有料プランにするバックエンドの処理（Edge Functionsなど）を実装することが必須です。

## 4. 個人事業主（Solo Developer）としてのベストプラクティス

リソース（時間・資金・労力）が限られている個人開発で成功するための戦略です。

1.  **MVP（Minimum Viable Product）を最速で出す**
    *   最初から「世界地図」を作るのではなく、まずは「一つの街（例えばカフェでの注文）」だけの体験版を作り、実際にユーザーに遊んでもらい反応を見ます。面白くなければ方向転換しやすくなります。
2.  **アセット（素材）を最大限活用する**
    *   3Dモデル、モーション、音楽などを全て自作すると完成しません。
    *   **推奨サイト**: Kenney.nl（無料・高品質な3Dモデル）、Unity Asset Store、Sketchfabなどを活用して時間を節約します。
    *   AI（ChatGPT, Claude, 生成系AI）を利用して、英語の例文作成やNPCの会話のバリエーションを生成するのも非常に有効です。
3.  **ランニングコストを抑える**
    *   ユーザーが少ない初期は、Vercel(無料枠), Supabase(無料枠), Cloudflareなどを組み合わせることで、**サーバー代・DB代を月額ほぼ0円（ドメイン代のみ）**で運用可能です。ユーザーが増え、収益が立つようになってから有料プランに移行します。
4.  **法務・規約面の整備**
    *   利用規約（ToS）、プライバシーポリシー、および**特定商取引法に基づく表記**（有料サービスを展開する日本国内の個人事業主には必須）をホームページに明記する必要があります（ひな形などを利用して作成可能です）。
5.  **マーケティングと発信**
    *   開発の初期段階から、X（旧Twitter）やTikTok、YouTube Shortsなどで「英語を教える宇宙人のゲーム作ってます」と開発過程を発信（Build in Public）することで、リリース時の初期ユーザーを獲得できます。
