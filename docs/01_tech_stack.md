# webブラウザゲームの技術スタックに関する詳細リサーチ（2024年最新版）

本ドキュメントでは、海外コミュニティ（Redditの`r/gamedev`など）の2023〜2024年のトレンドを踏まえ、トップダウン視点のWebゲーム（RPG風）を開発するための最適な技術スタックを深掘りします。

## 1. フロントエンド（ゲームエンジン編）
Webゲームエンジンにおける現在の三大会閥（Phaser, PixiJS, Godot）の比較です。

*   **Phaser 3 (JavaScript/TypeScript) - ★本プロジェクトの最有力候補**
    *   **Redditコミュニティの評価**: Webブラウザ向けの2Dゲーム開発において、最も頻繁に推奨される「全部入り」エンジン。物理演算（Arcade, Matter.js）、シーン管理、入力判定が最初から揃っています。
    *   **メリット**: Webネイティブであるためビルドサイズが非常に小さく、ブラウザでの起動が高速。子供向けアプリにおいて「ロード時間が長い」ことは致命的な離脱原因になるため、この点は非常に重要です。
    *   **デメリット**: ドキュメントがやや散らかっているという指摘や、最新エンジン（Godotなど）と比べるとエコシステムに古さを感じるという声もあります。
*   **PixiJS**
    *   **特徴**: ゲームエンジンというより「超高速な2Dレンダリングライブラリ」。Phaserも内部的にPixiJSの技術をベースにしています。
    *   **評価**: 描画パフォーマンスはPhaserの2倍とも言われますが、物理演算や当たり判定などを自作、または別のライブラリ（Matter.jsなど）と組み合わせる必要があるため、開発難易度が上がります。
*   **Godot 4 (GDScript / C#)**
    *   **特徴**: Unityに代わるオープンソースエンジンとして現在世界中で爆発的な人気を誇っています。
    *   **Web出力の課題**: Reddit等での2024年の議論では、「素晴らしいエンジンだが、Web（HTML5）エクスポートにはまだ課題がある」との声が多いです。特にGodot 4のWebビルドは初期サイズが約8MB〜と重く、iOSのSafari等でパフォーマンスが低下する（フレームレートが出ない）報告が多数あります。Web特化でいくならPhaserの方が安全です。

## 2. フロントエンド（UI・Webフレームワーク編）
ゲームキャンバス以外の部分（ログイン画面、親向けの学習ダッシュボード、決済画面など）の構築用です。

*   **React + Next.js (TypeScript)**
    *   ゲーム画面は `canvas` としてReactコンポーネント内にマウントし、それ以外のUIはReactで構築するのが現代のベストプラクティスです。Tailwind CSSを用いて、モダンで信頼感のある親向けUIを構築します。

## 3. バックエンド（BaaS - Backend as a Service）
学生の個人開発においては、インフラ管理の手間とコストを削るためにBaaSが必須です。

*   **Supabase (PostgreSQLベース) - ★推奨**
    *   Firebaseのオープンソース代替として現在大人気。リレーショナルデータベース（PostgreSQL）が使えるため、ユーザー、スコア、学習履歴、決済ステータスなどの複雑なデータ関係を安全に設計できます。
    *   **Row Level Security (RLS)** という強力なセキュリティ機能があり、「自分のデータしか読み書きできない」といった制限をデータベースレベルで簡単にかけられます。
*   **Firebase**
    *   NoSQL型のデータベース。手軽に始めるには良いですが、ゲームのデータ構造が複雑になるとクエリ（検索）が書きづらくなる欠点があります。

## 4. ホスティング・ドメイン・決済
*   **Vercel**: Next.jsとの相性が最高。Githubにプッシュするだけでゼロダウンタイムでデプロイされます。無料枠（Hobbyプラン）で最初は十分です。
*   **Cloudflare**: ドメイン取得（原価で買えるため最安）とCDN。DNSをCloudflareに向けることで、ゲームのアセット（画像や音声）を世界中へ高速に配信できます。
*   **Stripe**: サブスクリプション決済の業界標準。自社サーバーにクレジットカード情報を持たせることなく（PCI DSS準拠）、安全に月額ごとの自動課金システムを構築できます。
